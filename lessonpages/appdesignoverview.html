App Design Project - Overview

1 - Identify a User

May include surveys and research or a simple group decision on a target audience

2 - Identify User Needs

Research and analyze information about the potential user

3 - Paper Prototype

Students 'wire-frame' or layout the program on paper. Experience shows that students try to shortcut this important step. I dive into the Paper Prototype much like teaching how to build an outline or plan a writing.

4 - Testing and Research

This part can feel awkward, but once again is necessary. Teams work with other teams to do a blind walk-through of the paper prototype. A 'user' needs to try to explain how to use the program to the developer.

5 - Improve Paper Prototype

Standard process of revision. Teams take the information learned from testing and make improvements to the prototype.

6 - Build the UI (User Interface) for the Program

This step can be long and involve a few mini-lessons. It is safe to assume that I have already taught the basics of whatever code is necessary. At this point, we use the program as a chance to learn skills specific to a goal for the project. For example, we may build this program with the intent to learn: CSS classes, flex-box, divisions, etc.

7 - Code Events for the Program

Once again, this could be a long and involved step. It can also be a short process. Students just learning to code might need to apply basic events. A basic app or static website (like the one you are viewing) will have links and buttons. A more complicated program may require JavaScript for animation or data input/output.

Code Review
8 - Test and Feedback
9 - Improve and Retest(back to step 8)

A Code Review is much like the revision process you might see in ELA. Teams will need to allow for open testing of their program and let other developers look at their code to get feedback. Then students will 'refactor' their code (improve and hopefully look for efficiencies). Of course the process is not complete without retesting and getting more feedback.
10 - Publish

Pretty straightforward. The program is finally published for others to use.
